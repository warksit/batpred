---
name: Discuss before coding
description: Don't jump to code changes — discuss first, confirm changes are needed, answer questions
type: feedback
---

Don't just write code when the user asks a question. Often they're asking for understanding, not requesting a change.

**Why:** The user frequently asks diagnostic questions ("why is X happening?") to understand the system. Jumping straight to code changes without discussion leads to unnecessary fixes, missed root causes, and changes the user didn't ask for.

**How to apply:**
1. Answer the question first — explain what's happening and why
2. Ask if a change is needed before writing code
3. If a change IS needed, describe the approach and get agreement before implementing
4. Don't assume a question = a bug report
